# Project Architecture

## Overview
This project follows a modular backend architecture with a
clean separation of concerns.

## Backend Structure
- routes/       → API route definitions
- controllers/  → Request handling logic
- services/     → Core PDF processing logic
- middlewares/  → File upload and validation
- utils/        → Helper utilities

## Frontend
- Static HTML pages
- Internal CSS for simplicity
- JavaScript fetch-based API communication

## Design Principles
- Separation of concerns
- Stream-based file handling
- User-focused error handling
- Scalable structure for future growth
